【本草问诊 · 中医药辅助诊疗系统】2024.03 – 2024.06
项目负责人 | 独立开发（Claude Code AI Agent 辅助）

项目概述
基于 31 万余条真实中医医案数据，在 AI 编程 Agent（Claude Code）辅助下，独立完成了一款集智能问诊、节气养生、方剂检索、体质测评于一体的中医药辅助诊疗 Web 应用。实现双版本架构——专业版（基于知识库的规则推理）与 LLM 大模型版（DeepSeek 驱动），并部署至 Render 云平台提供公网访问。

技术架构
· 后端：Python Flask RESTful API，支持路由分离与双模诊断引擎自动降级；实现服务端按文件流式搜索（/api/search、/api/formula），避免 300MB 数据集全量加载导致内存溢出
· 前端：原生 HTML5 + CSS3 + JavaScript，绿色新中式 UI 风格，聊天式对话交互（打字动画、多轮上下文会话、自适应气泡布局）
· 部署：GitHub + Render 自动部署流水线，Git Push 即自动重新部署

核心功能
· 双模诊断引擎：LLM 模式集成 DeepSeek API（Chat Completions），定制中医辨证 System Prompt（八纲/脏腑/气血津液辨证框架、危险信号分级排查、方药安全提示与追问引导）；专业版基于症状模板库 + TCM 术语词典的模式匹配推理，在无 API 依赖时仍可正常服务
· 智能问诊对话：多轮追问式诊断，AI 自动清洗回复格式（移除 Markdown 标记、标题自动加粗放大）、支持快捷症状入口与对话历史管理
· 数据检索系统：按症状关键词、方剂名称搜索 31 万医案，返回证型诊断、方剂组成、中药剂量等结构化结果，支持收藏与导出
· 二十四节气养生：节气日历 + 智能推荐卡片（自动计算下一临近节气）、节气详情页（食疗方案、推荐食材、注意事项）
· 九种体质测评：60 题标准化问卷，自动计算体质倾向并生成报告（饮食原则、推荐食谱、养生建议）

项目亮点
· 借助 AI Agent 完成需求分析→架构设计→前后端编码→联调测试的全流程，将传统数周开发周期缩短至数天
· 设计按需加载与流式搜索策略，使 300MB 数据集在 Render 免费版（512MB 内存限制）上稳定运行
· 服务端统一管理 LLM API Key，用户端零配置即可使用大模型诊断，兼顾安全性与易用性
· 累计产出约 3,000 行代码，遵循 RESTful 规范，代码结构清晰、注释完备
