【中医药辅助诊疗系统】2024.03 – 2024.06
项目负责人 | 独立开发（AI Agent 辅助）

基于 31 万条真实中医医案数据，借助 AI 编程 Agent（Claude Code）辅助开发，高效构建了集智能问诊、节气养生、方剂检索于一体的中医药辅助诊疗 Web 应用。在 Agent 协同下完成需求分析→架构设计→前后端编码→联调测试的全流程迭代，累计产出约 1800 行代码。后端采用 Python Flask 搭建 RESTful API，加载 16 个分片 JSON 数据集（共 310,860 条医案），实现按症状关键词、性别、年龄的高效内存检索与病例统计分析；设计并实现了双模诊断引擎——LLM 模式（集成 DeepSeek API，定制中医辨证 System Prompt，实现危险信号排查、分型分析、穴位/方剂推荐、追问引导）与规则引擎模式（基于症状模板库+TCM 术语词典的离线诊断后备），确保系统在有无 API Key 时均可正常运行。前端基于原生 HTML/CSS/JavaScript 构建，采用绿色新中式 UI 风格，实现了二十四节气养生日历、智能节气推荐卡片、症状检索与方剂收藏、聊天式 AI 诊疗对话等完整功能模块。通过 Agent 驱动的 Skill 编排与多轮对话协作，将传统开发周期从数周缩短至数天，同时保持代码规范性与架构清晰度。
