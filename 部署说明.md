# 中医药辅助诊疗系统 — 部署说明

## 本地运行

```powershell
cd "E:\软件工程\大作业\中医药辅助诊疗系统2"
pip install -r requirements.txt
python app.py
```

打开浏览器访问：
- **规则引擎版**（无需 API Key）：http://127.0.0.1:5000/
- **LLM 大模型版**（需 DeepSeek Key）：http://127.0.0.1:5000/llm

---

## 部署到公网（免费 · Render）

### 1. 推送代码到 GitHub

```powershell
git add -A
git commit -m "中医药辅助诊疗系统 - 双版本"
git push origin main
```

### 2. 在 Render 部署

1. 打开 https://render.com → Sign Up（用 GitHub 登录）
2. 点 **New Web Service**
3. 连接你的 GitHub 仓库
4. Render 自动读取 `render.yaml` 配置，无需手动设置
5. 点 **Create Web Service**，等待 3-5 分钟部署完成

部署成功后会得到两个链接：
- `https://你的项目名.onrender.com/` → 规则引擎版（公开使用，无需 Key）
- `https://你的项目名.onrender.com/llm` → LLM 大模型版（服务端配 Key 后使用）

### 3. 配置 LLM（可选）

在 Render Dashboard → 你的服务 → Environment：
- 添加 `LLM_API_KEY` = `sk-你的DeepSeek Key`
- 点 Save Changes，服务自动重启

配置后两个版本都能使用大模型深度诊断。

---

## 两版说明

| | 规则引擎版 | LLM 大模型版 |
|---|---|---|
| 路径 | `/` | `/llm` |
| 需要 Key | ❌ 不需要 | ✅ 需要 |
| 诊断引擎 | 本地中医知识库 | DeepSeek 大模型 |
| 回复风格 | 结构化模板 | 千人千面个性化 |
| 适用场景 | 快速自查、科普 | 深度辨证分析 |
